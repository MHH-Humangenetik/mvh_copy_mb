# Commit Message Guidelines

When generating commit messages, follow these rules:

## Format

Use conventional commits format: `type(scope): description`

Common types:
- `feat`: new feature
- `fix`: bug fix
- `refactor`: code refactoring
- `docs`: documentation changes
- `test`: test additions or modifications
- `chore`: maintenance tasks
- `style`: formatting, styling changes
- `perf`: performance improvements

## Style

- Keep it to one sentence whenever possible
- Only add detailed explanations if the change is complex or non-obvious
- Add a suitable emoji at the start if it fits the change
- Be concise and direct
- Scope is optional but helpful for context
